# CoGibberTrue Design and Evolution Principles v0.1
UTC_CREATED: 20260306T201237Z

## Thesis
- CoGibberTrue should be concept-first, not word-first.
- Human-facing prose is a render layer, not the canonical meaning layer.

## Recommended layered model
1. CoSemanticKernel = canonical machine-native meaning layer
2. transport/syntax layer = stable serialized exchange form
3. CoRenderShell = human-facing translation / symbolic rendering

## Evolution rule
- evolution allowed, semantics anchored
- every meaningful artifact should carry enough version context to interpret its semantics correctly

## Version governance hints
- semantic version tag
- migration map
- deprecation markers
- token/context mapping

## Why this matters
- interpreting yesterday's CoGibberTrue with today's semantics can be dangerously wrong if the evolution protocol is unspecified
